# v0.4.8 — Schema Visualization & Table View Stability

**Release Date:** March 2026

Release focused on two areas: a new schema visualization canvas for exploring tables visually, and stability fixes for table-open query behavior. It also simplifies table-view cache state by removing redundant loaded flags.

## Completed Tasks

### Schema Visualization

- [x] Add `Open Table Visualize` to the sidebar table context menu
- [x] Add a dedicated `Visualize` tab type with one visualization tab per connection
- [x] Render the active schema/database as draggable table cards on a dotted canvas
- [x] Support pan, zoom, trackpad pinch-to-zoom, and reset view controls
- [x] Draw animated foreign-key relationship lines between related columns
- [x] Add schema visualization docs in `docs/schema-visualization.md`

### Table View Fetching

- [x] Fix duplicate initial `SELECT` requests when opening a table in production
- [x] Fix duplicate initial `COUNT(*)` requests caused by self-triggering table-view effects
- [x] Keep metadata fetches separate from paginated row fetches

### Table View State

- [x] Remove redundant `hasLoadedData`, `hasLoadedRowCount`, and `hasLoadedStructure` flags
- [x] Use cached `data`, `structure`, and nullable `totalRows` as the source of truth for load state
- [x] Preserve cached-tab restore behavior, including correct handling for empty tables
