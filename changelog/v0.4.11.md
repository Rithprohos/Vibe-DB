# v0.4.11 — Production Query Guardrails and Guided Action Policy

**Release Date:** March 2026

This release adds production-aware query guardrails to the query editor, keeps guided schema/data flows usable with confirmations, and documents the new behavior across the product docs.

## Completed Tasks

### Query Editor Production Guardrails

- [x] Block `DROP` and `TRUNCATE` in the query editor for production-tagged connections
- [x] Require explicit confirmation before `ALTER`, `CREATE`, `DELETE`, `INSERT`, `REPLACE`, and `UPDATE` run there
- [x] Support selected-SQL execution and multi-statement parsing for policy evaluation
- [x] Ignore semicolons inside strings and comments when splitting SQL statements
- [x] Show an in-editor production policy banner listing disabled statements

### Surface-Aware Enforcement

- [x] Pass connection environment tag metadata into the Rust backend connection registry
- [x] Add backend query-policy evaluation in Rust before engine execution
- [x] Distinguish between `query-editor` and guided app execution surfaces
- [x] Re-check blocked query-editor statements in Rust even if frontend checks already ran
- [x] Keep guided app flows usable instead of treating all SQL execution paths as ad hoc editor SQL

### Guided Destructive Actions

- [x] Refactor production row-delete confirmation to use a shared guided-mutation policy helper
- [x] Refactor truncate dialog copy and warnings to use the same guided-mutation policy helper
- [x] Preserve existing production confirmation behavior for destructive guided actions
- [x] Add "Drop Table" to sidebar context menu with production-aware confirmation dialog
- [x] Auto-close related tabs when a table is dropped
- [x] Show toast notifications for drop table success/error

### Documentation & Roadmap

- [x] Document the production query policy and guided-action split in `README.md`
- [x] Add roadmap follow-ups for per-user production guardrail overrides and bypass mode

### Tests

- [x] Add Rust unit tests for production query guard policy decisions
- [x] Add Rust coverage for guided-surface allowance behavior
- [x] Update engine registry tests for tagged connection config
- [x] Add engine tests for `drop_table` across SQLite and Turso engines
