# v0.4.1 — PostgreSQL Data Type UX Improvements

**Release Date:** March 2026

Focused PostgreSQL data-type quality release with better temporal/UUID decoding and a dedicated JSON/JSONB editing workflow.

## Completed Tasks

### PostgreSQL Type Decoding

- [x] Fix `TIMESTAMP`/`TIMESTAMPTZ`/`DATE`/`TIME` display pipeline by decoding native temporal values in Rust (instead of falling back to `NULL`)
- [x] Support PostgreSQL temporal aliases (for example `TIMESTAMP WITH TIME ZONE`, `TIME WITHOUT TIME ZONE`)
- [x] Fix native UUID decoding path and fallback handling so UUID values no longer render as `NULL`

### TableView & Inspector UX

- [x] Render JSON/JSONB values as JSON text instead of `[object Object]` in TableView/QueryEditor/Inspector displays
- [x] Add JSON-aware copy behavior from inspector/table fields
- [x] Add in-inspector field editing with save/cancel flow
- [x] Add JSON-aware editor helpers in inspector: live validity indicator, `Format`, and `Minify`
- [x] Add client-side JSON validation + smart-quote normalization before save to avoid PostgreSQL JSON syntax errors
- [x] Normalize JSON edits before staging so formatting-only changes do not create noisy pending edits

### Create/Edit Type Coverage

- [x] Expand PostgreSQL datatype options in Create/Edit table flows (numeric, temporal, UUID, JSON/JSONB, network, binary, serial variants)

### Documentation

- [x] Add PostgreSQL datatype support matrix in `ROADMAP.md`
- [x] Mark JSON/JSONB roadmap follow-up as completed and moved to changelog
