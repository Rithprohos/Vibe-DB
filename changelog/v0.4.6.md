# v0.4.6 — Backend SQL Generation & PG Type UX

**Release Date:** March 2026

Architecture hardening release completing the migration of all row mutation SQL to the Rust backend, eliminating frontend-built queries. Also ships parameterized PostgreSQL type support in the Create/Edit table UI.

## Completed Tasks

### Backend SQL Generation

- [x] Add `update_rows` command — Accept table name + column changes, build UPDATE queries in Rust
- [x] Add `insert_rows` command — Accept table name + row data, build INSERT queries in Rust
- [x] Deprecate frontend SQL builders (`buildWhereClause`, `buildDeleteQueries`, etc.)
- [x] Frontend becomes "UI only" — sends structured data to backend, receives results

### PostgreSQL Type UX

- [x] Parameterized types: support `VARCHAR(n)`, `NUMERIC(p,s)`, and temporal precision in Create/Edit table UI

### Performance

- [x] Debounce UI-triggered query refreshes
