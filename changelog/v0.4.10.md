**Release Date:** March 2026

This release hardens Create Table behavior across SQLite, Turso, and PostgreSQL, refactors the oversized Create Table UI into focused modules, and fixes a production persistence race where newly created connections could disappear after reload.

## Completed Tasks

### Connection Persistence Reliability

- [x] Serialize Zustand `tauri-plugin-store` writes to avoid overlapping save races
- [x] Add storage flush support and wait for persistence after successful connection setup
- [x] Fix production reload case where newly created connections could return users to the welcome screen

### Create Table Engine Support

- [x] Support engine-aware auto increment behavior in UI
- [x] Keep SQLite/Turso strict to `INTEGER PRIMARY KEY AUTOINCREMENT`
- [x] Support PostgreSQL serial mapping flows (`SMALLINT/INTEGER/BIGINT` to serial variants)
- [x] Add FK action controls for both `ON DELETE` and `ON UPDATE`

### Create Table Validation & UX

- [x] Add frontend validation for foreign key completeness and referenced local columns
- [x] Add frontend validation for check constraint expressions and optional constraint names
- [x] Block save with explicit inline errors instead of silently dropping incomplete constraints
- [x] Keep frontend and backend constraint validation behavior aligned

### Backend Constraint Safety

- [x] Update `build_create_table_sql` to reject partial foreign key constraints with clear errors
- [x] Update `build_create_table_sql` to reject check constraints missing expressions
- [x] Preserve valid named and unnamed check constraint SQL generation
- [x] Enable `PRAGMA foreign_keys = ON` on SQLite and Turso connect paths

### Refactor & Performance

- [x] Split `CreateTable.tsx` into focused modules (`ColumnRow`, `ColumnsSection`, `ConstraintsSection`, `validation`, `useCreateTableSqlPreview`)
- [x] Reduce `CreateTable.tsx` from monolithic layout to orchestrator-focused component
- [x] Gate SQL preview generation behind preview visibility to avoid unnecessary backend calls
- [x] Split frontend SQL utilities into focused modules by category (`identifiers`, `columnTypes`, `values`)
- [x] Split backend `sql_helpers` into focused Rust modules while preserving the existing command surface
- [x] Remove oversized catch-all SQL helper files in favor of responsibility-based structure

### Tooling & Standards

- [x] Upgrade the Tauri crate to Rust edition `2024`
- [x] Pin the Rust toolchain in `rust-toolchain.toml`
- [x] Update repo agent/docs guidance to require Rust `2024` only for backend code

### Tests

- [x] Add Rust coverage for FK action SQL generation
- [x] Add Rust coverage for partial FK validation errors
- [x] Add Rust coverage for empty check expression validation errors
- [x] Add Rust coverage for named check constraint SQL generation
